## Documentation Guidelines

### ОБЯЗАТЕЛЬНО

- **`README.md`** — краткое описание в каждом модуле.
- **`CHANGELOG.md`** — глобально в корне проекта.

### Не требуется

- Детальные описания каждого изменения  
- Документация по каждой функции  
- Отдельные `.md` файлы для фич, исправлений или модулей  
- Многоуровневая структура документации  
- Объемная документация по архитектуре или дизайну

### Формат CHANGELOG

Коротко добавляй одну строку на изменение:

```markdown
- YYYY-MM-DD: Краткое описание изменения
```
